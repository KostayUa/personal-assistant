# Специфікація вимог до системи (SRS)

**Проєкт:** Персональний помічник (CLI)  
**Версія:** 1.0
**Дата:** 2025-10-16

---

## 1. Огляд

**Персональний помічник** — це консольний застосунок, призначений для керування даними користувача, включаючи контакти
та нотатки.  
Додаток дозволяє створювати, редагувати, шукати та організовувати персональну інформацію за допомогою текстових команд
CLI.  
Основна мета — спростити управління персональною інформацією без використання графічного інтерфейсу.

---

## 2. Обсяг функціональності

1. **Управління контактами** — для зберігання та керування контактними даними.
2. **Управління нотатками** — для створення та організації текстових нотаток із опційними тегами.
3. Усі дані повинні зберігатися між запусками шляхом запису на диск користувача.

## 4. Модель взаємодії

### 4.1 Інтерфейс користувача

- Взаємодія через текстові команди CLI.
- Кожна операція виконується через конкретну команду.
- Система надає текстове повідомлення про успішне виконання або помилку.

### 4.2 Потік обробки команд

- Застосунок очікує введення команд у циклі.
- Парсить і перевіряє команди на коректність.
- Виконує операцію та показує результат.
- Цикл триває до введення команди `exit`.
- Команда `help` для виведення інформації по кожній команді. Приклад: `help add-contact`.

### 4.3 Обробка дат

- Дати повинні бути у форматі `dd.mm.yyyy`.
- Перевірка коректності враховує високосні роки; некоректні дати відхиляються з повідомленням про помилку.
- Валідація дати відбувається під час вводу.
- Обчислення "через N днів" коректно працює при переході через кінець року.

### 4.5 Команди

#### Додавання контакту

Додає контакт з усіма полями.

Команда: `add-contact ім'я [OPTIONS]`

Аргумент `ім'я`: ім'я контакту (унікальний).

Опційні ключі `[OPTIONS]`:

| Назва        | Опис                                                            | Приклад                                  |
|--------------|-----------------------------------------------------------------|------------------------------------------|
| `--phone`    | Ключ для вказання номеру телефону                               | `--phone "+380501234567"`                |
| `--phones`   | Ключ для вказання переліку номерів телефонів, розділених комами | `--phones "+380501111111,+380502222222"` |
| `--email`    | Ключ для вказання електронної пошти                             | `--email "ivan.petrenko@example.com"`    |
| `--birthday` | Ключ для вказання дати народження                               | `--birthday 16.10.1990`                  |
| `--address`  | Ключ для вказання адреси                                        | `--address "Київ, Україна"`              |

Примітки:

- ключі `--phone` та `--phones` не можуть використовуватись одночасно в одній команді.

Результат: інформація про те, що контакт було додано та повну інформацію, яка міститься у контакті.

#### Редагування контакту

Редагування даних контакту.

Команда: `edit-contact ім'я [OPTIONS]`

Аргумент `ім'я`: ім'я контакту (унікальний).

Опційні ключі `[OPTIONS]`:

| Назва            | Опис                                                             | Приклад                                      |
|------------------|------------------------------------------------------------------|----------------------------------------------|
| `--old-phone`    | Ключ для вказання старого номеру телефону                        | `--old-phone "+380501234567"`                |
| `--new-phone`    | Ключ для вказання нового номеру телефону                         | `--new-phone "+380501112233"`                |
| `--del-phone`    | Ключ для вказання номеру телефону, який треба видалити           | `--del-phone "+380501112233"`                |
| `--add-phone`    | Ключ для додавання нового номеру телефону                        | `--add-phone "+380501112233"`                |
| `--add-phones`   | Ключ для додавання переліку номерів телефонів, розділених комами | `--add-phones "+380501111111,+380502222222"` |
| `--email`        | Ключ для вказання електронної пошти                              | `--email "ivan.petrenko@example.com"`        |
| `--del-email`    | Ключ для вказівки того, що електронну пошту потрібно видалити    | `--del-email`                                |
| `--birthday`     | Ключ для вказання дати народження                                | `--birthday 16.10.1990`                      |
| `--del-birthday` | Ключ для вказівки того, що день народження потрібно видалити     | `--del-birthday`                             |
| `--address`      | Ключ для вказання адреси                                         | `--address "Київ, Україна"`                  |
| `--del-address`  | Ключ для вказівки того, що адресу потрібно видалити              | `--del-address`                              |

Примітки:

- ключі `--old-phone` та `--new-phone` використовуються тільки разом.
- ключі, які використовують для оновлення даних, не можуть застосовуватись із ключами, які видаляють ті ж самі дані.
- для додавання телефонів використовуйте `--add-phone` або `--add-phones`; додані телефони повинні бути унікальними в
  межах контакту (без урахування формату/пробілів), інакше — помилка.

Результат: інформація про те, що контакт було змінено та повну інформацію, яка тепер міститься у контакті.

#### Видалення контакту

Видаляє контакт.

Команда: `delete-contact ім'я [OPTIONS]`

Опційні ключі `[OPTIONS]`:

| Назва       | Опис                             | Приклад     |
|-------------|----------------------------------|-------------|
| `--confirm` | Ключ для підтвердження видалення | `--confirm` |

Примітки:

- без ключа `--confirm` система в інтерактивному режимі запросить підтвердження у користувача.

Результат: інформація про те, що контакт було видалено та повну інформацію, яка містилась у контакті.

#### Пошук контактів

Пошук контакту за наступними полями: ім'я, телефон, електронна пошта, дата народження.

Команда: `search-contact`

Опційні ключі `[OPTIONS]`:

| Назва        | Опис                                | Приклад                               |
|--------------|-------------------------------------|---------------------------------------|
| `--name`     | Ключ для вказання імені контакту    | `--name "Іван Петренко"`              |
| `--phone`    | Ключ для вказання номеру телефону   | `--phone "+380501234567"`             |
| `--email`    | Ключ для вказання електронної пошти | `--email "ivan.petrenko@example.com"` |
| `--birthday` | Ключ для вказання дати народження   | `--birthday 16.10.1990`               |

Примітки:

- під час виклику команди повинен бути вказаний щонайменше один із ключів;
- можна використовувати декілька ключів одночасно;
- правила співставлення:
    - name/email: пошук без урахування регістру;
    - phone: точний збіг після нормалізації до формату E.164 (ігноруючи пробіли/дефіси);
    - birthday: точний збіг;
- сортування результатів за іменем виконується без урахування регістру;
- якщо одночасно задано декілька ключів, то результати фільтруються з урахуванням усіх цих умов.

Результат:

- Список контактів відсортований на ім'я в алфавітному порядку, або
- повідомлення про те, що нічого незнайдено.

#### Перелік днів народження

Виводить контакти з днями народження протягом N днів.

Команда: `list-birthdays`

Опційні ключі `[OPTIONS]`:

| Назва    | Опис                             | Приклад    |
|----------|----------------------------------|------------|
| `--days` | Ключ для вказання кількості днів | `--days 7` |

Примітки:

- якщо ключ `--days` не вказаний, то використовується значення за замовчуванням котре дорівнює 7.

Результат:

- Перелік імен з датами народження, або
- повідомлення про те, що нічого незнайдено.

#### Додавання нотатки

Додає нотатку та теги до неї.

Команда: `add-note текст [OPTIONS]`

Аргумент `текст`: текст нотатки.

Опційні ключі `[OPTIONS]`:

| Назва    | Опис                            | Приклад                      |
|----------|---------------------------------|------------------------------|
| `--tags` | Ключ для вказання тегів нотатку | `--tags "зустріч,терміново"` |

Результат: інформація про те, що нотатка була додана та її ідентифікатор.

#### Додавання тегів

Додає теги до наявної нотатки.

Команда: `add-tags ідентифікатор теги`

Аргументи:

- `ідентифікатор` унікальний ідентифікатор нотатки.
- `теги` перелік тегів, розділених комами, котрі треба додати до нотатку.

Результат: інформація про те, що тег(и) були додані, та їх поточний повний список.

#### Видалення тегів

Видаляє теги з наявної нотатки

Команда: `delete-tags ідентифікатор теги [OPTIONS]`

Аргумент `ідентифікатор`: унікальний ідентифікатор нотатки.
Аргумент `теги`: перелік тегів, розділених комами, котрі треба видалити з нотатку.

Опційні ключі `[OPTIONS]`:

| Назва       | Опис                             | Приклад     |
|-------------|----------------------------------|-------------|
| `--confirm` | Ключ для підтвердження видалення | `--confirm` |

Примітки:

- без ключа `--confirm` система в інтерактивному режимі запросить підтвердження у користувача.
- якщо серед вказаних до видалення є теги, яких немає в нотатці, вони ігноруються; система виведе попередження про такі
  теги.

Результат: інформація про те, що тег(и) було видалено та перелік тегів, котрі залишились.

#### Редагування нотатки

Редагує текст і теги.

Команда: `edit-note ідентифікатор [OPTIONS]`

Аргумент `ідентифікатор`: унікальний ідентифікатор нотатки.

Опційні ключі `[OPTIONS]`:

| Назва        | Опис                                       | Приклад                          |
|--------------|--------------------------------------------|----------------------------------|
| `--text`     | Ключ для вказання нового тексту нотатку    | `--text "Зустріч о 10:00"`       |
| `--tags`     | Ключ для вказання тегів, які треба додати  | `--tags "зустріч,терміново"`     |
| `--del-tags` | Ключ для вказання тегів які треба видалити | `--del-tags "зустріч,терміново"` |

Примітки:

- під час виклику команди повинен бути вказаний щонайменше один із ключів;
- можна використовувати декілька ключів одночасно;
- якщо ключі `--tags` та `--del-tags` застосовуються разом в одній команді, то теги у значеннях для цих ключів повинні
  бути унікальні.

Результат: інформація про те, що текст та/або тег(и) було оновлено та повну інформацію, яка містилась у нотатку.

#### Вивести перелік нотаток

Команда: `list-notes [OPTIONS]`

Результат:

- Перелік нотаток, або
- повідомлення про те, що нічого незнайдено.

#### Вивести список унікальних тегів

Виводить перелік унікальних тегів, що використовуються у всіх нотатках.

Команда: `list-tags`

Результат:

- повідомлення про те, що нічого незнайдено.

#### Пошук нотаток

Пошук нотаток за частиною тексту та тегами.

Опційні ключі `[OPTIONS]`:

| Назва    | Опис                                       | Приклад                      |
|----------|--------------------------------------------|------------------------------|

| `--tags` | Ключ для вказання тегів нотатку            | `--tags "зустріч,терміново"` |

Примітки:

- під час виклику команди повинен бути вказаний щонайменше один із ключів;
- можна використовувати декілька ключів одночасно;
- правила пошуку:
    - текст: пошук підрядка без урахування регістру;
    - теги: порівняння без урахування регістру; збіг за будь-яким із вказаних тегів (OR);
- якщо одночасно задано `--text` і `--tags`, результати фільтруються обома умовами.

Результат:

- Перелік нотаток, або
- повідомлення про те, що нічого незнайдено.

Видаляє нотатку.

## 8. Модель даних та унікальність

### Contact (Контакт)

Ключ/унікальність: ім'я є унікальним ключем (без урахування регістру). Дублікати імен не допускаються.

| Поле     | Тип               | Обов'язковість |
|----------|-------------------|----------------|
| name     | string            | обов'язково    |
| phones   | array of phone    | опційно        |
| email    | string            | опційно        |
| address  | string            | опційно        |
| birthday | date `dd.mm.yyyy` | опційно        |

### Note (Нотатка)

Ключ/унікальність: нотатка зберігається за своїм `id`.

| Поле | Тип                      | Обов'язковість |
|------|--------------------------|----------------|
| id   | унікальний ідентифікатор | обов'язково    |
| text | string                   | обов'язково    |
| tags | array of tag             | опційно        |

Примітка: Використання унікального імені для контактів спрощує CLI, але потребує чітких повідомлень про конфлікти при
додаванні/перейменуванні.

## 9. Правила валідації вводу

- Телефон
    - Підтримка міжнародного формату E.164: `+` і до 15 цифр. Дозволяється введення з пробілами/дефісами, що
      нормалізуються при зберіганні.
    - Приклади валідних: `+380501234567`, `+1 650-555-0100` (нормалізується до `+16505550100`).
- Email
    - Базова перевірка: один символ `@`, домен з принаймні однією крапкою, лише ASCII/IDN-домен за потреби. Приклад:
      `user.name+tag@example.com` — валідний.
- Дата народження
    - Формат суворо `dd.mm.yyyy`, перевірка високосних років, виключення неіснуючих дат (напр., 31.11.2024 — невалідна).
- Теги нотаток
    - Ввід через роздільник кома: `--tags "робота,терміново"`.
    - Нормалізація: обрізання пробілів навколо; теги при зберіганні нормалізуються до нижнього регістру.
    - Порівняння та унікальність тегів — без урахування регістру (case-insensitive).
    - Обмеження: до 20 тегів на нотатку, довжина одного тегу — до 30 символів, при перевищенні — відмова з
      повідомленням.
- Обмеження розмірів:
    - Текст нотатки — до 500 символів, при перевищенні — відмова з повідомленням.
    - Ім'я контакту — до 32 символів, при перевищенні — відмова з повідомленням.
    - Адреса — до 300 символів, при перевищенні — відмова з повідомленням.
